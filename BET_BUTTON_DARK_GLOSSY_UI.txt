✅ FEATURE 1: DYNAMIC BET/CALL BUTTON


Problem:
  The "CALL" button was always showing, even when there was no bet to call.
  In poker, when no one has bet yet, you should see "BET" instead of "CALL".


Solution Implemented:
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  WHEN currentBet === 0:
  ┌────────────────────────────────────┐
  │                                    │
  │         🟣  BET  🟣                │  <- Purple/Magenta glow
  │                                    │
  │  Use this to start the betting     │
  │  Makes initial bet (raise amount)  │
  └────────────────────────────────────┘
  
  Button Style:
  • Background: purple-900/60 → purple-700
  • Text: purple-300 → white
  • Glow: Purple shadow (rgba(168,85,247,0.3))
  • Hover glow: Stronger (rgba(168,85,247,0.5))


  WHEN currentBet > 0:
  ┌────────────────────────────────────┐
  │                                    │
  │         🔵  CALL  🔵               │  <- Blue (original)
  │                                    │
  │  Use this to match the bet         │
  └────────────────────────────────────┘
  
  Button Style:
  • Background: blue-900/50 → blue-800
  • Text: blue-400 → white
  • Standard appearance


  Code Changes:
  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  
  1. App.tsx:
     - Pass currentBet prop to Actions component:
       <Actions currentBet={currentBet} ... />
  
  2. Actions.tsx:
     - Accept currentBet prop (default 0)
     - Conditional rendering:
       {currentBet === 0 ? (
         <button className="...purple..." onClick={() => onAction('raise', raiseAmount)}>
           BET
         </button>
       ) : (
         <button className="...blue..." onClick={() => onAction('call')}>
           CALL
         </button>
       )}


  Visual Comparison:

  [FOLD]                          [FOLD]
  [CHECK]                         [CHECK]
  [CALL] ← Always blue            [BET] ← Purple when currentBet = 0
  [RAISE]                         [RAISE]
  [ALL IN]                        [ALL IN]
  [LEAVE]                         [LEAVE]

                                  [CHECK]

💎 DESIGN NOTES


Why This Works:
  
  1. BET vs CALL clarity:
     • Different colors help distinguish game state
     • Purple = initiating action (aggressive)
     • Blue = responding to action (passive)
     • Matches poker terminology exactly
  
  2. Ultra-dark aesthetic:
     • Reduces eye strain for long sessions
     • Makes neon accents pop more
     • Creates premium, professional feel
     • Better focus on important elements
  
  3. Glossy effects:
     • Multiple shadow layers = depth perception
     • Shine gradients = polished surface
     • Inset shadows = recessed panels
     • Creates "expensive" appearance
  
  4. Color psychology:
     • Dark blue = Trust, stability, depth
     • Black = Power, elegance, mystery
     • Purple = Royalty, luxury (for BET)
     • Cyan accents = Energy, technology
