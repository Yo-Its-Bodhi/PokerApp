  Cards were returning color as 'text-red-500' or 'text-white' (CSS classes)
  but Card.tsx was expecting plain strings 'red' or 'black'
  
  This caused the inline style={{ color: ... }} to fail:
  ❌ style={{ color: 'text-red-500' }} → Invalid CSS, card text invisible!

THE FIX:
  Updated cardToString() in Table.tsx:
  
  BEFORE (BROKEN):
  const color = (suit === '♥' || suit === '♦') ? 'text-red-500' : 'text-white';
  
  AFTER (FIXED):
  const color = (suit === '♥' || suit === '♦') ? 'red' : 'black';
  
  Now Card.tsx receives:
  ✅ style={{ color: 'red' }}      → Valid CSS, red text!
  ✅ style={{ color: 'black' }}    → Valid CSS, black text!

BUTTON LOGIC BY SITUATION:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

A) NO BET TO YOU (toCall == 0)
   
   Show: [CHECK] [BET amount] [ALL IN]
   
   • Check is free action (primary button)
   • Bet with slider: min = minBet, max = stack
   • If stack < minBet: Only show ALL IN
   
   Example:
   ┌─────────────────────────────────────────────┐
   │  [  CHECK  ] [BET 5,000] [  ALL IN  ]       │
   └─────────────────────────────────────────────┘

B) FACING A BET (toCall > 0)
   
   Show: [FOLD] [CALL amount] [RAISE to amount] [ALL IN]
   
   • Fold to forfeit hand
   • Call shows exact amount to call
   • Raise if can meet minRaiseTo
   • If stack <= toCall: Show ALL IN instead of CALL
   
   Example (normal stack):
   ┌──────────────────────────────────────────────────┐
   │  [FOLD] [CALL 5,000] [RAISE TO 15,000] [ALL IN] │
   └──────────────────────────────────────────────────┘
   
   Example (short stack):
   ┌─────────────────────────────────────┐
   │  [  FOLD  ] [ALL IN 3,000]         │
   └─────────────────────────────────────┘

C) PLAYER ALL-IN
   
   Show: No buttons, just status badge
   
   ┌─────────────────────────────────┐
   │     🔥 ALL IN - WAITING 🔥      │
   └─────────────────────────────────┘

  Bets reset to 0

EDGE CASES HANDLED:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Short-Raise All-In
   Player shoves < minRaiseTo
   → Shows: [ALL IN amount] with note "Short-raise; may not reopen"
   → Engine enforces no re-raise from earlier players

2. Exact Call All-In
   toCall >= stack
   → Shows: [ALL IN stack] instead of [CALL]
   → No raise option available

3. Bet Below Min
   Player tries to bet < minBet
   → Validation returns: { valid: false, adjustedAmount: minBet }
   → UI can auto-adjust or show error

4. Raise Below Min
   Player tries to raise < minRaiseTo
   → Validation returns: { valid: false, adjustedAmount: minRaiseTo }
   → UI enforces minimum

5. Over-Bet Protection
   Player tries to bet > stack
   → Validation returns: { valid: true, adjustedAmount: stack }
   → Caps at all-in amount
